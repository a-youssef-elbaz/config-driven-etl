STEP 7: Create the Prompt Templates (KEY STEP)
─────────────────────────────────────────────────────────
This is where AI helps you build the AI instructions.

Ask Copilot:
  "@workspace I need to create two prompt templates for this
   repository. Read #file:.github/copilot-instructions.md
   for context about what this repo does.

   PROMPT 1 (prompts/us-to-lld.md):
   Write a prompt template that instructs an AI to read a
   user story file and produce a Low-Level Design document.
   The LLD must include:
   - Source-to-target column mapping
   - Join specifications (type, condition, column selection)
   - Transform specifications (pass-through mode, computed cols)
   - Output mapping (final column list with aliases)
   - Merge key identification
   The AI should reference #file:docs/CONFIG_FILLING_GUIDE.md
   for table schemas and #file:docs/Config_Examples.sql for
   correct patterns.

   PROMPT 2 (prompts/lld-to-config.md):
   Write a prompt template that instructs an AI to read an
   LLD document and produce:
   - A CONFIG SQL file (INSERT statements in dependency order)
   - A ROLLBACK SQL file (DELETE statements in reverse order)
   The AI should follow the 9-step Config Filling Order.
   If new expressions are needed, generate them separately
   for shared/expressions.sql.
   Reference #file:docs/Config_Examples.sql for exact syntax."

Review Copilot's output carefully. Iterate until both
prompts produce good results when tested (Step 9).

Save to: prompts/us-to-lld.md and prompts/lld-to-config.md

STEP 9: Generate LLD + CONFIG + ROLLBACK (First Real Test)
─────────────────────────────────────────────────────────
This is where you test the entire Copilot workflow.

Ask Copilot:
  "@workspace Using #file:prompts/us-to-lld.md, generate
   an LLD for #file:products/kk/US-kk-base-load.md.
   Reference #file:docs/CONFIG_FILLING_GUIDE.md for schemas
   and #file:docs/Config_Examples.sql for patterns."

Review the LLD. Does it correctly identify:
  - All source tables?
  - Join types and conditions?
  - Transform logic?
  - Output column mapping?
  - Merge keys?

If not, refine and ask Copilot to fix specific parts.
Save to: products/kk/LLD-kk-base-load.md

Then continue:
  "@workspace Using #file:prompts/lld-to-config.md, generate
   CONFIG and ROLLBACK SQL for the LLD you just created.
   Reference #file:docs/Config_Examples.sql for exact syntax."

Review the CONFIG. Compare it against the SQL that's
ALREADY working in BigQuery for this rule. They should
produce equivalent configs.

Save to: products/kk/CONFIG-kk-base-load.sql
         products/kk/ROLLBACK-kk-base-load.sql


STEP 10: Validate Your Reference Example
─────────────────────────────────────────────────────────
Compare the generated CONFIG against what already exists
in BigQuery. Run in BQ console:

  SELECT * FROM ds_metadata_config.t_pbmis_rule_config
  WHERE rule_id = 'KK_BASE_LOAD';

Does the generated INSERT match the existing row?
Repeat for all config tables.

If there are differences, refine your prompts (Step 7)
and regenerate (Step 9). Iterate until the Copilot output
matches your existing production config.

This iteration is the most important part of Phase 1.
The prompts are your "compiler" — get them right and
every future rule generation will be consistent.

STEP 13: Set Up Branch Protection Rules
─────────────────────────────────────────────────────────
On github.com → Settings → Branches → Add rule:

For each branch (Develop, sit, uat, preprod, main):
  ✅ Require pull request before merging
  ✅ Required approvals: 2
  ✅ Require status checks to pass (add later in Phase 2)
  ❌ Allow force pushes: OFF
  ❌ Allow deletions: OFF
